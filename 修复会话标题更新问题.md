# 修复会话标题更新问题

## 问题描述

在 AI 诊断页面，新建对话后，与 AI 对话后，左边的会话标签总是显示"新建会话"，没有根据会话内容自动更新标题。

## 问题原因

1. **前后端标题不一致**：前端创建会话时使用中文"新建会话"，但后端检查的是英文"New Session"
2. **条件判断失败**：由于标题不匹配，后端的标题更新逻辑永远不会触发
3. **前端未刷新**：即使后端更新了标题，前端也没有在对话完成后刷新会话列表

## 修复方案

### 1. 后端修改 (backend/routers/chat.py)

#### 修改 1：支持中英文标题检查

**位置**：第 168-178 行

**修改前：**
```python
if session:
    if session.title == "New Session":
        session.title = user_message[:80]
    if model_id and not session.ai_model_id:
        session.ai_model_id = model_id
```

**修改后：**
```python
if session:
    # Update title if it's still the default (support both Chinese and English)
    if session.title in ("New Session", "新建会话"):
        session.title = user_message[:80] if user_message else "[Attachment]"
    if model_id and not session.ai_model_id:
        session.ai_model_id = model_id
```

**改进点：**
- 同时支持中文"新建会话"和英文"New Session"的检查
- 处理只有附件没有文本的情况
- 截取前 80 个字符作为标题，避免过长

#### 修改 2：清除消息时使用中文标题

**位置**：第 117-124 行

**修改前：**
```python
session = result.scalar_one_or_none()
if session:
    session.title = "New Session"
```

**修改后：**
```python
session = result.scalar_one_or_none()
if session:
    session.title = "新建会话"
```

**改进点：**
- 与前端保持一致，使用中文"新建会话"

### 2. 前端修改 (frontend/js/pages/diagnosis.js)

#### 修改：对话完成后刷新会话列表

**位置**：第 582-600 行

**修改前：**
```javascript
case 'done':
    ChatWidget.finishAssistantMessage();
    break;
```

**修改后：**
```javascript
case 'done':
    ChatWidget.finishAssistantMessage();
    // Refresh session list to update title after first message
    this._loadSessions();
    break;
```

**改进点：**
- 在对话完成后自动刷新会话列表
- 确保更新后的标题立即显示在左侧边栏

## 工作流程

修复后的完整流程：

1. **用户创建新会话**
   - 前端调用 API 创建会话，标题为"新建会话"
   - 会话显示在左侧边栏

2. **用户发送第一条消息**
   - 前端通过 WebSocket 发送消息
   - 后端接收消息并保存

3. **后端更新会话标题**
   - 检查会话标题是否为"新建会话"或"New Session"
   - 如果是，则使用用户消息的前 80 个字符更新标题
   - 保存到数据库

4. **AI 响应完成**
   - 后端发送 `done` 事件
   - 前端接收到 `done` 事件后调用 `_loadSessions()`
   - 重新加载会话列表，显示更新后的标题

5. **用户看到更新后的标题**
   - 左侧边栏的会话标签显示为用户消息的内容（前 80 字符）
   - 不再显示"新建会话"

## 测试验证

### 测试步骤

1. 启动应用：`python run.py`
2. 打开浏览器，进入 AI 诊断页面
3. 点击"New"按钮创建新会话
4. 发送一条消息，例如："查看数据库状态"
5. 等待 AI 响应完成
6. 检查左侧边栏的会话标签

### 预期结果

- 会话标签应该从"新建会话"变为"查看数据库状态"
- 如果消息超过 80 个字符，应该只显示前 80 个字符
- 后续消息不会再改变标题

### 边界情况测试

1. **只发送附件**：标题应该显示为"[Attachment]"
2. **发送很长的消息**：标题应该截断为 80 个字符
3. **清除会话消息**：标题应该重置为"新建会话"
4. **切换会话**：每个会话的标题应该独立更新

## 注意事项

1. **向后兼容**：修改同时支持中文和英文标题，不会影响现有数据
2. **性能影响**：每次对话完成后会刷新会话列表，但由于会话数量通常不多，性能影响可忽略
3. **标题长度**：限制为 80 个字符，避免标题过长影响显示
4. **数据库字段**：确保 `DiagnosticSession.title` 字段长度足够（建议至少 200 字符）

- `frontend/js/pages/diagnosis.js` - 前端诊断页面逻辑

## 总结

本次修复解决了会话标题不更新的问题，主要通过以下三个改进：

1. 后端支持中英文标题检查
2. 后端正确处理附件消息的标题
3. 前端在对话完成后自动刷新会话列表

修复后，用户体验得到显著改善，会话标签能够准确反映对话内容，方便用户识别和管理多个会话。
