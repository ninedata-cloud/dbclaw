# AI诊断界面重新设计

## 设计概述

重新设计了AI诊断页面的交互界面，使工具调用和结果的展示更加清晰和易用。

## 主要改进

### 1. 三栏布局
- **左侧栏**: 会话列表（280px）
- **中间区域**: 对话消息流（自适应）
- **右侧栏**: 工具执行面板（400px，可折叠）

### 2. 工具执行面板
工具调用从消息流中分离出来，在右侧独立面板展示：

#### 特性：
- **实时状态显示**:
  - 🔵 Running（运行中）- 蓝色，带旋转动画
  - 🟢 Complete（完成）- 绿色
  - 🔴 Error（错误）- 红色

- **可折叠卡片**:
  - 点击工具卡片头部可展开/折叠
  - 展开后显示参数和结果
  - 默认折叠，节省空间

- **详细信息**:
  - **Arguments**: 显示工具调用的参数（JSON格式）
  - **Result**: 显示工具执行结果（JSON或文本格式）

- **面板控制**:
  - 折叠/展开整个工具面板
  - 清空工具历史记录

### 3. 视觉优化

#### 工具卡片样式：
- 圆角卡片设计，带阴影
- 状态标签带背景色
- 代码内容区域使用等宽字体
- 自定义滚动条样式

#### 动画效果：
- 工具卡片展开/折叠动画
- 运行状态旋转动画
- 面板折叠平滑过渡

### 4. 用户体验改进

- **清晰的信息层次**: 对话内容和工具执行分离，互不干扰
- **实时反馈**: 工具执行状态实时更新
- **可控性**: 用户可以选择查看或隐藏工具详情
- **空间利用**: 工具面板可折叠，最大化对话区域

## 使用说明

### 查看工具执行：
1. 发送消息后，AI调用的工具会实时显示在右侧面板
2. 点击工具卡片可展开查看参数和结果
3. 工具状态会实时更新（运行中 → 完成/错误）

### 管理工具面板：
- 点击 `⊟` 按钮折叠/展开整个工具面板
- 点击 `🗑️` 按钮清空工具历史记录
- 工具按时间倒序排列（最新的在顶部）

## 示例场景

当AI执行数据库诊断时：
1. 用户在中间区域看到AI的文字回复
2. 右侧面板实时显示AI调用的工具：
   - `execute_sql` - 执行SQL查询
   - `get_table_info` - 获取表信息
   - `analyze_slow_query` - 分析慢查询
3. 点击任意工具卡片可查看具体的SQL语句和执行结果
4. 所有信息清晰分离，易于追踪和调试
