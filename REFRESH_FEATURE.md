# 性能监控刷新功能

## 功能说明

在性能监控界面右上角新增了"刷新"按钮，点击后可以立即触发一次监控数据采集并展示到前端。

## 实现细节

### 后端实现

1. **新增API端点**: `POST /api/metrics/{conn_id}/refresh`
   - 位置: `backend/routers/metrics.py`
   - 功能: 手动触发指定数据源的指标采集
   - 验证: 检查数据源是否存在且已激活
   - 调用: `metric_collector.collect_metrics_for_connection(conn_id)`

### 前端实现

1. **API客户端方法**: `API.refreshMetrics(connId)`
   - 位置: `frontend/js/api.js`
   - 功能: 调用后端刷新API

2. **刷新按钮**:
   - 位置: `frontend/js/pages/monitor.js`
   - 样式: 蓝色主按钮，带刷新图标
   - 交互:
     - 点击时禁用按钮并显示"采集中..."状态
     - 调用API触发采集
     - 等待1秒确保数据已存储
     - 重新加载最新数据
     - 恢复按钮状态

3. **CSS样式**:
   - 位置: `frontend/css/monitor.css`
   - 样式: 按钮布局、禁用状态、图标大小

## 使用方法

1. 打开性能监控页面
2. 选择要监控的数据源
3. 点击右上角的"刷新"按钮
4. 系统会立即采集一次监控数据并更新显示

## 技术特点

- **异步处理**: 使用async/await确保不阻塞UI
- **状态反馈**: 按钮状态变化提供清晰的用户反馈
- **错误处理**: 采集失败时显示错误提示
- **数据同步**: 采集后自动刷新最新数据到前端
