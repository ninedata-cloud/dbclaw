# 修复 report_generator.py 导入错误

## 问题描述

在生成巡检报告时出现以下错误：

```
NameError: name 'REPORT_GENERATION_PROMPT' is not defined
```

## 根本原因

`backend/services/report_generator.py` 文件中使用了 `REPORT_GENERATION_PROMPT` 常量，但没有导入该常量。

## 修复内容

在 `backend/services/report_generator.py` 文件开头添加了所有缺失的导入：

```python
import logging
from datetime import datetime

from sqlalchemy import select

from backend.models.datasource import Datasource
from backend.models.report import Report
from backend.models.host import Host
from backend.agent.prompts import REPORT_GENERATION_PROMPT
from backend.database import async_session
from backend.utils.encryption import decrypt_value
from backend.services.db_connector import get_connector
from backend.services.ssh_service import SSHService
from backend.services.os_metrics_service import OSMetricsService
from backend.agent.conversation_skills import generate_report_with_skills
```

## 修复的导入项

1. **REPORT_GENERATION_PROMPT** - AI 报告生成的系统提示词
2. **Host** - 主机模型
3. **async_session** - 异步数据库会话
4. **decrypt_value** - 密码解密函数
5. **get_connector** - 数据库连接器工厂函数
6. **SSHService** - SSH 服务
7. **OSMetricsService** - 操作系统指标服务
8. **generate_report_with_skills** - 使用技能生成报告的函数

## 验证

已通过导入测试验证修复成功：
- ✓ REPORT_GENERATION_PROMPT 导入成功
- ✓ report_generator 模块导入成功

## 影响范围

此修复解决了智能巡检功能中报告生成失败的问题。现在可以正常：
1. 手动触发巡检
2. 定时自动巡检
3. 异常触发巡检
4. 生成 AI 分析报告
