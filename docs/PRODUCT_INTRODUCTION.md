# DBGuard - AI 驱动的智能数据库运维平台

## 产品概述

DBGuard 是新一代 AI 驱动的数据库运维平台，为企业提供全方位的数据库健康管理解决方案。通过融合人工智能、自动化运维和实时监控技术，DBGuard 帮助 DBA 团队从被动响应转向主动预防，显著提升数据库系统的稳定性和性能。

### 核心价值

- **智能诊断**：基于大语言模型的自然语言交互，快速定位数据库性能瓶颈和故障根因
- **主动监控**：实时采集关键指标，智能阈值告警，第一时间发现潜在问题
- **自动巡检**：定时执行健康检查，生成专业诊断报告，防患于未然
- **多库支持**：统一管理 MySQL、PostgreSQL、Oracle、SQL Server、国产数据库等 10+ 种数据库
- **可扩展架构**：86+ 内置诊断技能，支持自定义技能和第三方系统集成

## 产品特性

### 1. AI 智能诊断引擎

DBGuard 集成先进的大语言模型，提供类似 ChatGPT 的自然语言交互体验，让数据库诊断变得简单直观。

**核心能力**：
- **自然语言查询**：用日常语言提问，如"为什么数据库这么慢？"、"哪些 SQL 占用资源最多？"
- **意图识别**：自动识别诊断、查询、管理三种意图，匹配最优系统提示词
- **上下文理解**：结合历史对话、实时指标、知识库内容，提供精准分析
- **根因分析**：深度分析性能问题，给出可执行的优化建议
- **多轮对话**：支持连续追问，逐步深入问题细节

**技术亮点**：
- 动态技能选择：AI 自动从 86+ 技能中选择最合适的诊断工具
- 知识库增强：支持导入企业内部运维文档，提升诊断准确性
- 网络搜索集成：遇到未知问题可实时搜索最新解决方案

### 2. 全方位监控体系

**实时指标采集**：
- **数据库层指标**：连接数、QPS/TPS、慢查询、锁等待、缓存命中率、表空间使用率
- **操作系统指标**：CPU、内存、磁盘 I/O、网络流量（通过 SSH 采集）
- **灵活采集方式**：
  - 系统内置采集：直连数据库实时查询
  - 适配器采集：对接 Prometheus、Zabbix 等第三方监控系统
- **可配置间隔**：支持按数据源重要性设置不同采集频率

**智能告警系统**：
- **多维度阈值**：支持静态阈值、动态基线、复合条件
- **告警聚合**：5 分钟时间窗口内相同问题自动合并，减少告警风暴
- **告警去重**：60 分钟去重窗口，避免重复通知
- **自动恢复**：指标恢复正常后自动解除告警
- **多渠道通知**：Webhook、钉钉、邮件、短信（可扩展）
- **订阅管理**：按数据源、严重级别、时间段灵活配置通知规则

**可视化监控**：
- **实时仪表盘**：WebSocket 推送，秒级刷新关键指标
- **历史趋势图**：支持自定义时间范围查看指标变化
- **告警中心**：统一查看、确认、处理所有告警事件

### 3. 自动巡检服务

定时执行全面健康检查，生成专业诊断报告，让运维工作从被动响应转向主动预防。

**巡检能力**：
- **定时调度**：支持按天、周、月配置巡检计划
- **阈值检查**：自动检测 CPU、内存、磁盘、连接数等关键指标
- **AI 分析**：可选启用 AI 深度分析，提供优化建议
- **报告生成**：自动生成结构化巡检报告，包含问题汇总和处理建议
- **触发方式**：
  - 定时触发：按配置的时间间隔自动执行
  - 阈值触发：指标超限时自动触发巡检
  - 手动触发：随时发起临时巡检

**报告内容**：
- 数据源基本信息和配置
- 关键指标当前值和历史趋势
- 发现的问题和风险点
- AI 生成的优化建议
- 可执行的操作步骤

### 4. 多数据库统一管理

**支持的数据库类型**（10+ 种）：
- **开源数据库**：MySQL、PostgreSQL、MongoDB、Redis
- **商业数据库**：Oracle、SQL Server
- **国产数据库**：达梦（DM）、openGauss
- **分布式数据库**：TiDB、OceanBase

**统一管理能力**：
- 集中配置：统一管理所有数据源连接信息
- 安全加密：数据库密码采用 Fernet 加密存储
- SSH 隧道：支持通过跳板机访问内网数据库
- 连接池：复用 SSH 连接，提升性能
- 连接测试：配置前验证连接可用性

### 5. 可编程适配器系统

灵活对接第三方监控系统，实现监控数据的统一汇聚和分析。

**适配器特性**：
- **完全可编程**：在 Web 界面直接编写 Python 代码
- **无沙箱限制**：可导入任意 Python 模块，调用任何 API
- **自动调度**：按配置的时间间隔自动执行数据采集
- **批量写入**：高效的指标批量入库机制
- **实时生效**：修改代码后立即生效，无需重启服务

**典型应用场景**：
- 对接 Prometheus：拉取已有监控指标
- 对接 Zabbix：复用现有监控体系
- 对接云厂商 API：采集 RDS 监控数据
- 自定义采集：实现特殊业务指标采集

### 6. 技能系统架构

DBGuard 采用插件化的技能系统，所有诊断能力都以技能形式组织，易于扩展和维护。

**技能特性**：
- **86+ 内置技能**：覆盖各数据库的常见诊断场景
- **YAML 定义**：声明式配置，易于理解和修改
- **参数验证**：支持类型检查、范围验证、正则匹配
- **权限控制**：细粒度权限管理（查询、命令、日志读取等）
- **沙箱执行**：安全隔离，防止恶意代码
- **超时控制**：可配置执行超时（默认 30 秒，最大 300 秒）
- **审计日志**：记录所有技能执行历史

**技能分类**：
- **数据库诊断**：慢查询分析、锁等待检测、表空间检查、索引使用率
- **性能分析**：执行计划分析、等待事件统计、复制状态检查
- **系统管理**：连接管理、会话查询、配置查看
- **操作系统**：CPU/内存/磁盘监控、进程管理
- **通知集成**：邮件、短信、Webhook 发送
- **数据查询**：监控数据查询、巡检报告查询、知识库搜索

**技能示例**：
```yaml
id: mysql_get_slow_queries
name: MySQL 慢查询分析
category: performance
parameters:
  - name: connection_id
    type: integer
    required: true
  - name: limit
    type: integer
    default: 10
    min: 1
    max: 100
permissions:
  - execute_query
timeout: 30
code: |
  async def execute(context, params):
      conn = await context.get_connection(params['connection_id'])
      query = "SELECT * FROM mysql.slow_log ORDER BY query_time DESC LIMIT %s"
      result = await context.execute_query(query, params['connection_id'], [params['limit']])
      return {"slow_queries": result}
```

## 应用场景

### 场景 1：快速故障诊断

**问题**：凌晨 3 点，业务系统突然变慢，用户投诉激增。

**解决方案**：
1. 打开 DBGuard AI 诊断页面
2. 输入："生产库为什么突然变慢了？"
3. AI 自动执行：
   - 检查当前连接数和活跃会话
   - 分析最近的慢查询
   - 检查锁等待情况
   - 查看系统资源使用率
4. 给出诊断结论："发现 3 个长时间运行的查询导致锁等待，建议 kill 掉这些会话"
5. DBA 确认后执行，问题解决

**价值**：从发现问题到定位根因，从 30 分钟缩短到 3 分钟。

### 场景 2：主动预防故障

**问题**：希望在问题发生前就发现潜在风险。

**解决方案**：
1. 配置自动巡检：每天凌晨 2 点执行
2. 设置阈值规则：
   - CPU 使用率 > 80% 持续 5 分钟
   - 磁盘使用率 > 85%
   - 连接数 > 最大连接数的 80%
3. 启用 AI 分析：自动生成优化建议
4. 配置告警订阅：钉钉群接收巡检报告

**价值**：每天自动发现 5-10 个潜在问题，避免 80% 的故障发生。

### 场景 3：多数据库统一运维

**问题**：企业有 50+ 个数据库实例（MySQL、PostgreSQL、Oracle 混合），运维复杂。

**解决方案**：
1. 在 DBGuard 中统一添加所有数据源
2. 配置分级监控：
   - 核心库：30 秒采集一次
   - 普通库：60 秒采集一次
   - 测试库：300 秒采集一次
3. 设置统一告警规则
4. 通过仪表盘一屏查看所有数据库健康状态

**价值**：运维效率提升 3 倍，告警响应时间从 10 分钟降到 1 分钟。

### 场景 4：对接现有监控体系

**问题**：已有 Prometheus 监控系统，不想重复采集数据。

**解决方案**：
1. 创建 Prometheus 适配器
2. 编写采集代码（30 行 Python）：
```python
async def fetch_metrics(datasources):
    metrics = []
    for ds in datasources:
        # 从 Prometheus 查询指标
        response = await http_client.get(
            f"{prometheus_url}/api/v1/query",
            params={"query": f'mysql_global_status_threads_connected{{instance="{ds["host"]}"}}'}
        )
        value = response['data']['result'][0]['value'][1]
        metrics.append({
            "datasource_id": ds["id"],
            "metric_name": "connections",
            "metric_value": float(value),
            "timestamp": datetime.now()
        })
    return metrics
```
3. 设置 60 秒采集间隔
4. DBGuard 自动将数据入库，供 AI 诊断使用

**价值**：复用现有监控数据，无需重复建设，快速获得 AI 诊断能力。

## 部署方案

### 单机部署（推荐用于测试/小规模生产）

**环境要求**：
- 操作系统：Linux / macOS / Windows
- Python：3.10+
- PostgreSQL：12+
- 内存：4GB+
- 磁盘：20GB+

**部署步骤**：
```bash
# 1. 克隆代码
git clone https://github.com/your-org/dbguard.git
cd dbguard

# 2. 安装依赖
pip install -r requirements.txt

# 3. 配置环境变量
cp .env.example .env
# 编辑 .env，设置数据库连接、AI API Key 等

# 4. 初始化数据库
# PostgreSQL 会在首次启动时自动初始化

# 5. 启动服务
python run.py

# 6. 访问系统
# 浏览器打开 http://localhost:9939
```

