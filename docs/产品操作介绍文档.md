# DBClaw 产品操作介绍文档

适用产品：NineData DBClaw（数据库智能卫士）

适用读者：DBA、SRE、运维工程师、平台管理员、实施工程师和安全审批人员。

本文介绍 DBClaw 从部署访问、初始化配置、资源接入、监控诊断、告警巡检到系统管理的完整操作流程。实际界面名称以当前版本 Web 控制台为准。

## 一、产品简介

DBClaw 是一款 AI 原生的数据库智能运维平台，面向私有化部署场景，提供数据库统一纳管、实时监控、AI 对话诊断、自动巡检、告警通知、知识库增强和外部系统集成能力。

DBClaw 当前支持以下数据库类型：

- MySQL
- PostgreSQL
- Oracle
- SQL Server
- openGauss
- SAP HANA

产品主要面向以下使用场景：

- 数据库实例统一接入、分组、标签化管理。
- 数据库指标持续采集与资源大盘展示。
- 数据库异常、慢 SQL、连接、锁等待、参数等问题的 AI 辅助诊断。
- 数据库和主机联动分析，定位数据库问题背后的系统资源瓶颈。
- 定时巡检和事件触发巡检，形成结构化巡检报告。
- 告警事件管理、通知分发、静默、确认和解决闭环。
- 通过知识库、技能系统和 AI 模型配置增强诊断效果。
- 对接飞书、钉钉、企业微信、Webhook、邮件和云厂商监控指标。

## 二、访问与登录

### 2.1 访问地址

服务启动后，在浏览器访问：

```text
http://<服务器地址>:9939
```

本机部署时默认地址为：

```text
http://127.0.0.1:9939
```

### 2.2 默认管理员账号

首次启动时系统会自动创建默认管理员账号：

```text
用户名：admin
默认密码：admin1234
```

如果部署时配置了 `INITIAL_ADMIN_PASSWORD`，默认密码以该环境变量为准。

首次登录后建议立即完成以下安全操作：

- 修改管理员默认密码。
- 创建个人或团队账号，避免多人共用 `admin`。
- 检查系统密钥、数据库连接串和公网访问边界。
- 生产环境通过 HTTPS 或反向代理保护管理入口。

### 2.3 退出登录与个人信息

登录后，左侧导航底部显示当前用户信息。点击用户区域可执行：

- 修改个人资料：维护显示名称、邮箱、电话等信息。
- 修改密码：输入当前密码和新密码后保存。
- 退出登录：结束当前登录会话。

## 三、首次初始化建议流程

首次部署后，建议按以下顺序完成初始化：

1. 登录系统并修改默认管理员密码。
2. 进入“AI 大模型管理”，添加至少一个可用 AI 模型。
3. 进入“系统参数配置”，检查告警、巡检、通知、云厂商等基础参数。
4. 进入“主机管理”，添加需要关联监控的服务器。
5. 进入“数据源管理”，添加数据库实例并测试连接。
6. 为核心数据源配置巡检周期、告警模板和静默策略。
7. 进入“任务调度管理”，启用关键任务并验证运行历史。
8. 进入“知识库”，导入内部运维手册、故障案例和处理规范。
9. 进入“外部集成管理”，配置通知渠道、云厂商指标或机器人。
10. 进入“资源大盘”和“实例详情”确认监控数据是否正常刷新。

AI 模型未配置时，数据源管理、监控、巡检、告警等基础能力仍可使用，但 AI 对话、AI 诊断和 AI 判警能力不可用或效果受限。

## 四、界面导航说明

DBClaw Web 控制台采用左侧导航布局，主要功能包括：

- 资源大盘：查看数据库与主机资源总览、告警概况、运行状态。
- 实例详情：查看单个数据库实例的监控、会话、SQL、告警、巡检和参数。
- 主机详情：查看单台主机的监控、进程、终端、网络拓扑和 AI 诊断。
- 智能巡检：查看巡检报告、触发来源、巡检结果和历史记录。
- 告警管理：查看告警事件、告警明细、通知订阅和告警模板。
- AI 大模型管理：维护 AI 模型连接、启用状态和测试配置。
- 技能管理：查看、启用、测试和维护 AI 可调用的诊断技能。
- 知识库：维护文档分类、知识文档和 AI 诊断上下文。
- 数据源管理：新增、编辑、测试和删除数据库连接。
- 主机管理：新增、编辑、测试和删除 SSH 主机连接。
- 外部集成管理：配置通知、入站指标、机器人等外部系统。
- 任务调度管理：统一管理系统内周期任务、执行状态和运行历史。
- 系统参数配置：维护全局系统参数。
- 用户管理：管理员维护用户、密码、启用状态和登录日志。

普通用户和管理员看到的导航可能不同，用户管理等能力通常仅管理员可见。

## 五、AI 大模型管理

AI 大模型是 DBClaw 进行对话式诊断、巡检分析、AI 判警和报告总结的基础能力。

### 5.1 新建模型

操作入口：

```text
左侧导航 -> AI 大模型管理 -> 新建 AI 模型
```

常见配置项包括：

- 模型名称：便于在系统内识别，例如 `DeepSeekV4`、`Qwen3.6-plus`、`GPT-5.5`。
- API Base URL：模型服务的 OpenAI 兼容接口地址。
- API Key：模型服务访问密钥。
- 模型标识：实际调用的模型名称。
- 启用状态：控制模型是否可被系统选择。

保存后建议使用“测试模型”功能验证模型可用性。

### 5.2 模型选择规则

系统使用模型时通常按以下优先级选择：

1. 当前会话指定的模型。
2. 系统中第一个启用的模型。
3. 环境变量中的 `OPENAI_*` 兜底配置。

建议在 Web 控制台统一维护模型配置，环境变量仅作为兼容或兜底方案。

### 5.3 操作建议

- 生产环境建议配置稳定、低延迟、可审计的企业模型服务。
- 至少保留一个可用启用模型，避免 AI 功能不可用。
- 模型密钥属于敏感信息，应限制后台访问权限。
- 如诊断结果不稳定，可优先检查模型可用性、上下文长度和知识库质量。

## 六、数据源管理

数据源是 DBClaw 进行数据库监控、巡检、SQL 查询和 AI 诊断的核心对象。

### 6.1 新建数据源

操作入口：

```text
左侧导航 -> 数据源管理 -> 新建数据源
```

填写信息：

- 数据源名称：建议使用业务系统、环境和实例标识，例如 `prod-order-mysql-01`。
- 数据库类型：选择 MySQL、PostgreSQL、Oracle、SQL Server、TDSQL-C MySQL、openGauss 或 SAP HANA。
- 主机地址和端口：填写数据库访问地址和端口。
- 用户名和密码：建议使用专用只读监控账号。
- 数据库名：按数据库类型和接入方式填写，可为空。
- 标签：用逗号分隔，例如 `生产, 订单, 核心系统`。
- 备注：填写业务背景、特殊架构、读写角色等信息，AI 诊断时会自动带入。
- 关联主机：可选择已接入的主机，用于数据库与操作系统指标联动分析。
- 重要性级别：选择核心系统、生产系统、开发测试或临时。
- 监控数据来源：选择系统采集或集成采集。

保存后，点击“测试连接”确认数据库连通性。

### 6.2 监控数据来源

DBClaw 支持两种监控数据来源：

- 系统采集：DBClaw 直连数据库，周期性采集数据库运行状态。
- 集成采集：通过外部集成系统拉取云厂商或第三方监控指标。

选择集成采集时，需要先在“外部集成管理”中创建并启用 `入站指标` 类型集成，然后在数据源中选择对应集成，并填写外部实例 ID 和采集参数。

### 6.3 数据库账号权限建议

建议为 DBClaw 创建专用监控账号：

- 默认使用只读账号，不使用超级管理员账号。
- 授予读取业务对象、系统视图、性能视图和元数据的权限。
- 默认不授予 DDL、DML、DCL 权限。
- 默认不授予终止会话权限，确需使用时单独审批。

账号授权脚本可参考：

- `docs/客户交付版-数据库账号与权限清单.md`
- `docs/实施手册版-数据库账号与权限配置说明.md`

### 6.4 数据源常用操作

在数据源列表中可执行：

- 查看实例详情：进入单实例工作台。
- 编辑数据源：修改连接信息、标签、备注和监控来源。
- 立即诊断：跳转到 AI 诊断并带入当前数据源上下文。
- 测试连接：验证数据库连通性和账号可用性。
- 巡检与告警配置：配置当前数据源巡检计划和告警模板。
- 设置告警静默：临时暂停该数据源告警通知。
- 删除数据源：移除不再纳管的实例。

删除数据源前应确认不再需要历史监控、巡检和告警关联信息。

## 七、主机管理

主机用于采集操作系统指标，并为数据库诊断提供 CPU、内存、磁盘、网络、进程和终端能力。

### 7.1 新建主机

操作入口：

```text
左侧导航 -> 主机管理 -> 新建主机
```

填写信息通常包括：

- 主机名称：便于识别的名称。
- 主机地址：服务器 IP 或域名。
- SSH 端口：默认通常为 `22`。
- 用户名和认证信息：按现场 SSH 策略填写。
- 备注：记录用途、业务系统、环境等信息。

保存前建议使用测试连接功能验证 SSH 连通性。

### 7.2 关联数据库实例

在“数据源管理”中新建或编辑数据源时，可选择“关联主机”。关联后，DBClaw 可在实例详情和 AI 诊断中同时使用数据库指标与主机指标。

建议优先为以下实例配置关联主机：

- 核心生产数据库。
- 性能问题频发的数据库。
- 需要分析 CPU、内存、磁盘 IO、网络 IO 的数据库。
- 需要通过 Web 终端进行排障的服务器。

### 7.3 主机详情

操作入口：

```text
左侧导航 -> 主机详情
```

主机详情通常包含：

- 基本信息：主机配置、状态、关联数据源等。
- 性能监控：CPU、内存、磁盘、网络等指标。
- 实时进程：查看进程资源占用。
- Terminal：通过浏览器访问主机终端。
- 网络拓扑：查看主机网络连接关系。
- AI 诊断：结合主机指标发起智能排障。

Web 终端能力涉及较高权限，生产环境应限制可访问用户并保留审计。

## 八、资源大盘

资源大盘用于快速查看整体运行状态。

操作入口：

```text
左侧导航 -> 资源大盘
```

常见信息包括：

- 数据库实例总数与运行状态。
- 主机总数与运行状态。
- 当前活跃告警。
- 近期告警趋势。
- 关键资源或实例状态概览。

建议日常值班时优先查看资源大盘，发现异常后再进入实例详情、主机详情或告警管理进一步处理。

## 九、实例详情

实例详情是单个数据库实例的主要操作工作台。

操作入口：

```text
左侧导航 -> 实例详情
```

也可以从“数据源管理”列表点击“实例详情”进入。

### 9.1 实例基本信息

展示数据源的连接信息、类型、标签、备注、关联主机、重要性级别、告警摘要和巡检配置等。

常见操作：

- 测试连接。
- 查看实例状态。
- 进入巡检或告警相关配置。
- 发起 AI 诊断。

### 9.2 性能监控

性能监控展示数据库关键指标趋势，例如：

- 连接数。
- QPS / TPS。
- 慢 SQL 或 TOP SQL 相关指标。
- 缓存、锁、等待、会话等数据库状态。
- 关联主机的 CPU、内存、磁盘、网络指标。

可根据时间范围切换查看最近 1 分钟、10 分钟、1 小时、1 天、7 天或自定义时间范围。

### 9.3 流量拓扑

流量拓扑用于观察客户端到数据库实例的访问关系和网络流量，帮助判断访问来源、流量变化和异常连接。

常见用途：

- 识别异常客户端。
- 查看入站和出站流量趋势。
- 分析数据库访问链路。
- 辅助定位突增流量来源。

### 9.4 实时会话

实时会话用于查看当前数据库连接和活动会话。

可用于：

- 查看活跃 SQL。
- 分析长事务、锁等待和高耗时请求。
- 识别异常连接来源。
- 在具备权限时终止会话。

终止会话属于高风险操作，建议仅在明确影响和获得授权后执行。

### 9.5 AI 对话诊断

AI 对话诊断会带入当前实例上下文，结合监控指标、会话、知识库和技能工具进行分析。

推荐提问方式：

```text
这个实例最近 CPU 升高的原因是什么？
```

```text
帮我分析当前连接数异常增长是否和某类 SQL 有关。
```

```text
根据最近一小时指标，判断这个实例是否存在锁等待或慢 SQL 风险。
```

为了获得更好效果，建议在数据源备注中补充业务背景，例如实例角色、业务系统、读写分离架构、维护窗口等。

### 9.6 SQL 窗口

SQL 窗口提供 Web 化 SQL 查询能力。

常见操作：

- 选择数据源。
- 选择数据库或 Schema。
- 编写并执行 SQL。
- 查看查询结果。
- 查看查询历史。

安全建议：

- 使用只读账号接入生产库。
- 不建议在生产环境使用高权限账号执行 SQL。
- 对可能影响业务的 SQL，应先在低峰期或测试环境验证。

### 9.7 TOP SQL

TOP SQL 用于查看高耗时、高频率或高资源消耗 SQL。

常见排查方向：

- SQL 执行次数是否异常。
- 平均耗时是否升高。
- 是否存在全表扫描或索引缺失。
- 是否与告警或性能指标波动时间一致。

不同数据库需要对应系统视图或扩展权限，例如 PostgreSQL 常依赖 `pg_stat_statements`。

### 9.8 告警管理

实例内的告警管理展示当前实例相关告警。

可执行：

- 查看告警事件。
- 查看告警明细和上下文。
- 确认告警。
- 标记解决。
- 设置静默。
- 跳转 AI 诊断。

### 9.9 巡检管理

实例内巡检管理用于查看当前实例巡检配置和巡检结果。

可执行：

- 查看巡检报告。
- 手动触发巡检。
- 调整巡检周期。
- 查看基线与异常项。

### 9.10 实例参数配置

实例参数配置用于查看数据库运行参数，辅助判断参数是否合理。

常见使用方式：

- 搜索关键参数。
- 对比参数值与建议值。
- 将参数异常作为 AI 诊断上下文。
- 结合巡检报告判断配置风险。

## 十、性能监控

除实例详情内的性能监控外，系统也提供独立的“性能监控”页面能力，用于按数据源查看指标历史和健康状态。

常见操作：

- 选择数据源。
- 选择时间范围。
- 查看最新指标快照。
- 查看历史指标趋势。
- 分析指标异常时间点。

监控数据默认按系统配置的采集间隔持续写入。若图表长时间无数据，请优先检查：

- 数据源连接是否成功。
- 数据库账号权限是否足够。
- 后台服务是否正常运行。
- 数据源监控数据来源是否配置正确。
- 外部集成采集是否启用且参数正确。

## 十一、AI 诊断

AI 诊断是 DBClaw 的核心能力之一，可通过自然语言与系统交互，辅助完成问题分析、SQL 排查、指标解释和处理建议生成。

操作入口：

```text
左侧导航 -> AI 诊断
```

也可以从数据源、告警、巡检报告或实例详情进入，并自动带入上下文。

### 11.1 新建会话

新建会话时可选择：

- 关联数据源。
- 使用的 AI 模型。
- 会话标题。
- 技能授权范围。

建议每个独立问题创建单独会话，便于后续追踪和复盘。

### 11.2 诊断上下文

AI 诊断可结合以下上下文：

- 数据源基础信息。
- 最近指标和告警。
- 巡检报告。
- 知识库文档。
- 技能执行结果。
- 用户上传或输入的补充信息。

### 11.3 技能授权

AI 可以调用已启用的技能执行诊断查询或系统检查。技能可能访问数据库、主机或外部工具，因此应按最小权限原则授权。

建议：

- 普通排障优先授权只读查询和诊断类技能。
- 高风险技能需谨慎启用，并结合审批流程。
- 对生产环境终止会话、执行命令类能力保持严格控制。

### 11.4 审批与高风险操作

当 AI 需要执行高风险工具或需要用户确认的操作时，系统会触发审批流程。用户应查看操作说明、目标对象和影响范围后再决定是否允许。

## 十二、智能巡检

智能巡检用于定期或按事件触发检查数据库健康状态，并生成结构化报告。

操作入口：

```text
左侧导航 -> 智能巡检
```

### 12.1 巡检触发方式

巡检可由以下方式触发：

- 定时触发：按数据源配置的周期自动执行。
- 手动触发：用户在实例详情或巡检页面主动执行。
- 异常触发：监控指标或告警事件触发。
- 阈值触发：达到配置的阈值规则后触发。

系统存在触发去重窗口，用于避免同一问题短时间内重复生成巡检任务。

### 12.2 查看巡检报告

巡检报告通常包含：

- 巡检对象。
- 触发方式。
- 巡检时间。
- 总体结论。
- 异常项和风险项。
- 指标摘要。
- AI 分析与处理建议。

可进入报告详情查看具体检查项、上下文和建议。

### 12.3 巡检配置

在数据源的“巡检与告警配置”中可配置：

- 是否启用自动巡检。
- 巡检周期，例如每小时、每 6 小时、每天、每周、每月。
- 告警模板。
- 基线和异常判断配置。

建议核心生产库启用定时巡检，并结合告警事件触发巡检，形成日常健康检查和异常复盘闭环。

## 十三、告警管理

告警管理用于处理数据库和主机异常，支持事件聚合、通知订阅、告警模板、确认、解决和静默。

操作入口：

```text
左侧导航 -> 告警管理
```

### 13.1 告警事件

告警事件表示一次可追踪的异常过程。一个事件下可能包含多条告警记录。

常见操作：

- 按状态、严重级别、数据源、时间范围筛选。
- 查看事件详情。
- 查看关联告警和上下文。
- 确认事件。
- 标记解决。
- 设置静默。
- 发起 AI 诊断。

### 13.2 告警状态处理

建议告警处理流程：

1. 查看告警事件详情，确认影响范围。
2. 查看关联指标和上下文。
3. 必要时进入实例详情、主机详情或 AI 诊断。
4. 判断是否需要立即处理、观察、静默或升级。
5. 处理完成后标记解决。
6. 将根因和处理方式沉淀到知识库。

### 13.3 告警静默

告警静默适用于维护窗口、已知变更、临时压测等场景。

静默前应确认：

- 静默对象是否正确。
- 静默时间范围是否合理。
- 是否存在仍需通知的严重告警。
- 维护结束后是否需要取消静默。

### 13.4 告警模板

告警模板用于定义阈值规则、严重级别、通知策略和 AI 判警策略。

常见配置内容包括：

- 模板名称。
- 适用对象。
- 告警级别。
- 指标阈值。
- 通知渠道。
- AI 判警模式。

系统支持阈值告警和 AI 判警。AI 判警可结合置信度、冷却期和连续确认次数减少误报。

### 13.5 通知订阅

通知订阅用于配置告警发送目标，例如：

- Webhook。
- 邮件。
- 飞书。
- 钉钉。
- 企业微信。

配置通知前，应先在“外部集成管理”中创建对应出站通知集成。

## 十四、知识库

- 处理步骤。
- 回滚方案。

### 14.3 使用建议

- 对生产变更流程、应急联系人、升级路径等保持更新。

## 十九、典型操作场景

### 19.1 接入一个新的生产数据库

推荐流程：

1. DBA 创建 DBClaw 专用只读监控账号。
2. 管理员在“主机管理”中添加数据库所在主机。
3. 管理员在“数据源管理”中新建数据源。
4. 选择数据库类型，填写地址、端口、账号、密码和数据库名。
5. 选择关联主机、重要性级别和标签。
6. 点击“测试连接”。
7. 进入“巡检与告警配置”，设置巡检周期和告警模板。
8. 进入“实例详情”确认监控数据正常。
9. 发起一次手动巡检，保存首份基线报告。

### 19.2 处理一次数据库性能告警

推荐流程：

1. 在“告警管理”中打开告警事件。
2. 查看告警级别、触发时间、数据源和指标上下文。
3. 点击进入实例详情，查看性能监控和实时会话。
4. 查看 TOP SQL 和流量拓扑，判断是否存在异常 SQL 或客户端。
5. 如有关联主机，进入主机详情查看 CPU、内存、磁盘和网络。
6. 点击 AI 诊断，让系统结合上下文生成分析。
7. 根据建议执行处理动作。
8. 告警恢复后标记解决。
9. 将根因和处理步骤记录到知识库。

### 19.3 做一次月度数据库巡检

推荐流程：

1. 在“智能巡检”或实例详情中筛选目标数据源。
2. 对核心实例手动触发巡检，或确认定时巡检已完成。
3. 查看巡检报告中的异常项、风险项和建议。
4. 对高风险项发起 AI 诊断或转派 DBA 处理。
5. 整理报告结论并归档。
6. 将常见问题沉淀为知识库文档或巡检规范。

### 19.4 配置告警通知到 IM 群

推荐流程：

1. 在“外部集成管理”中新建出站通知集成。
2. 选择飞书、钉钉、企业微信或 Webhook。
3. 填写机器人地址、Token 或必要参数。
4. 使用测试功能确认消息可发送。
5. 在“告警管理”中配置通知订阅或告警模板。
6. 触发测试告警或等待实际告警验证推送效果。

### 19.5 使用 AI 分析巡检报告

推荐流程：

1. 打开“智能巡检”中的报告详情。
2. 从报告详情跳转 AI 诊断，或新建 AI 会话并指定报告上下文。
3. 提问异常项原因、影响范围和修复建议。
4. 对建议中的 SQL、参数、命令进行人工确认。
5. 将确认后的处理方案沉淀到知识库。

### 19.6 配置任务调度并验证执行

推荐流程：

1. 进入“任务调度管理”，点击“新建任务”。
2. 填写任务名称、任务类型和执行说明，便于后续维护。
3. 选择调度方式：Cron 表达式或固定间隔（秒）。
4. 绑定失败通知渠道（如 Webhook、飞书、钉钉、企业微信或邮件）。
5. 保存后手动点击“立即执行”，验证任务配置和权限是否正确。
6. 在“运行历史”中检查执行结果、耗时和错误信息。
7. 将关键生产任务保持“已启用”状态，并定期检查最近执行状态。

## 二十、运维与安全建议

### 20.1 部署安全

- 生产环境必须修改默认管理员密码。
- 使用强随机 `ENCRYPTION_KEY` 和 `PUBLIC_SHARE_SECRET_KEY`。
- 使用 HTTPS 或反向代理保护 Web 控制台。
- 限制 DBClaw 到元数据库、目标数据库和主机 SSH 的网络访问范围。
- 定期备份 PostgreSQL 元数据库、`/app/data` 和 `/app/uploads`。

### 20.2 数据库接入安全

- 使用专用只读监控账号。
- 避免使用 `root`、`sys`、`sa`、`sysdba` 等超级账号。
- 终止会话、系统命令和高风险技能需单独审批。
- 对生产库 SQL 查询保持审计意识。

### 20.3 AI 使用安全

- 不在对话中输入不必要的敏感信息。
- 对 AI 建议进行人工复核后再执行。
- 限制高风险技能授权范围。
- 定期维护知识库，避免过期规范影响诊断结论。

### 20.4 日常维护

- 定期检查数据源连接状态。
- 定期检查主机 SSH 连接状态。
- 定期检查告警通知渠道是否可用。
- 定期检查 AI 模型是否可用。
- 定期归档巡检报告和故障复盘。
- 定期清理无效数据源、主机和用户。

## 二十一、常见问题

### 21.1 登录后看不到用户管理

用户管理通常仅管理员可见。请确认当前账号是否具备管理员权限。

### 21.2 AI 诊断不可用

请检查：

- 是否已配置并启用 AI 模型。
- 模型 API Key 是否正确。
- API Base URL 是否可访问。
- 模型测试是否成功。
- 当前会话是否选择了可用模型。

### 21.3 数据源连接失败

请检查：

- 数据库地址和端口是否正确。
- DBClaw 服务器到数据库网络是否连通。
- 用户名和密码是否正确。
- 数据库账号是否允许远程访问。
- 数据库账号权限是否足够。
- Oracle、SQL Server、HANA 等数据库是否需要额外连接参数。

### 21.4 监控图表没有数据

请检查：

- 数据源测试连接是否成功。
- 指标采集服务是否正常运行。
- 采集周期是否合理。
- 数据源是否选择了正确的监控数据来源。
- 外部集成是否启用并能正常拉取指标。

### 21.5 巡检报告没有 AI 总结

请检查：

- AI 模型是否可用。
- 巡检任务是否成功执行。
- 数据源是否有足够监控数据。
- 知识库或技能调用是否发生错误。

### 21.6 告警没有发送通知

请检查：

- 告警模板或通知订阅是否启用。
- 外部通知集成是否配置正确。
- Webhook、邮件或 IM 机器人测试是否成功。
- 告警是否处于静默时间段。
- 告警是否被聚合、冷却或自动恢复。

### 21.7 主机终端无法打开

请检查：

- 主机 SSH 连接是否测试成功。
- SSH 端口是否被防火墙拦截。
- 用户认证方式是否正确。
- 当前账号是否具备访问终端的权限。
- 后端 WebSocket 服务是否正常。

### 21.8 任务调度没有按计划执行

请检查：

- 任务是否处于“已启用”状态。
- Cron 表达式或固定间隔配置是否正确。
- 服务器时间与时区配置是否符合预期。
- 任务依赖的数据源、集成或权限是否可用。
- 最近运行历史中的错误信息是否提示连接失败或参数错误。

任务调度管理用于集中配置和管理系统中的周期性任务，适用于巡检触发、指标同步、通知分发和其他自动化运维动作。
