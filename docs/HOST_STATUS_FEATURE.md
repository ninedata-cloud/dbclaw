# 主机状态显示功能实现文档

## 功能概述

为主机管理列表添加了实时状态显示功能，可以直观地查看每台主机的运行状态和资源使用情况。

## 状态类型

### 1. 正常 (Normal)
- **显示**: ✓ 正常（绿色徽章）
- **条件**:
  - CPU 使用率 < 80%
  - 内存使用率 < 80%
  - 磁盘使用率 < 80%
  - 最近5分钟内有数据更新

### 2. 异常 (Warning)
- **显示**: ⚠ 异常（黄色徽章）
- **条件**:
  - CPU 使用率 >= 80% 且 < 90%
  - 或 内存使用率 >= 80% 且 < 90%
  - 或 磁盘使用率 >= 80% 且 < 90%
- **提示信息**: 显示具体的异常项（如"CPU使用率较高 (85.2%)"）

### 3. 严重 (Error)
- **显示**: ✗ 严重（红色徽章）
- **条件**:
  - CPU 使用率 >= 90%
  - 或 内存使用率 >= 90%
  - 或 磁盘使用率 >= 90%
- **提示信息**: 显示具体的严重问题（如"CPU使用率过高 (95.8%)"）

### 4. 离线 (Offline)
- **显示**: ○ 离线（灰色徽章）
- **条件**:
  - 超过5分钟未收到监控数据
  - 或 从未收到过监控数据
- **提示信息**: "连接失败（超过5分钟未收到数据）" 或 "暂无监控数据"

## 显示内容

### 表格列
1. **名称** - 主机名称
2. **主机** - IP 地址
3. **端口** - SSH 端口
4. **用户名** - SSH 用户名
5. **状态** - 状态徽章（带悬停提示）
6. **CPU** - CPU 使用率（带颜色标识）
7. **内存** - 内存使用率（带颜色标识）
8. **磁盘** - 磁盘使用率（带颜色标识）
9. **操作** - 测试/编辑/删除按钮

### 指标颜色标识
- **正常** (< 80%): 默认颜色
- **警告** (80-89%): 黄色
- **危险** (>= 90%): 红色

## 实现细节

### 后端实现

#### 1. Schema 更新 (`backend/schemas/host.py`)
```python
class HostResponse(BaseModel):
    # ... 原有字段
    cpu_usage: Optional[float] = None
    memory_usage: Optional[float] = None
    disk_usage: Optional[float] = None
    status: str = "unknown"  # normal, warning, error, offline
    status_message: Optional[str] = None
    last_check_time: Optional[datetime] = None
```

#### 2. API 更新 (`backend/routers/hosts.py`)
- 从 `host_metrics` 表获取最新指标
- 计算指标新鲜度（5分钟阈值）
- 根据指标值判断状态级别
- 生成状态消息

**状态判断逻辑**:
```python
# 检查数据新鲜度
if metric_age > 300:  # 5分钟
    status = "offline"
else:
    # 检查指标阈值
    if any(metric >= 90):
        status = "error"
    elif any(metric >= 80):
        status = "warning"
    else:
        status = "normal"
```

### 前端实现

#### 1. 表格渲染 (`frontend/js/pages/hosts.js`)
- 新增 `_renderHostRow()` 方法
- 根据状态显示不同颜色的徽章
- 指标值根据阈值显示不同颜色
- 状态徽章支持悬停显示详细信息

#### 2. 样式支持 (`frontend/css/main.css`)
- 新增 `badge-secondary` 样式（灰色）
- 新增文本颜色类：
  - `.text-success` - 绿色
  - `.text-warning` - 黄色
  - `.text-danger` - 红色
  - `.text-info` - 蓝色
  - `.text-muted` - 灰色

## 数据流程

```
1. host_collector.py (后台任务)
   ↓ 每60秒收集一次
2. host_metrics 表
   ↓ 存储时序数据
3. /api/hosts (API)
   ↓ 查询最新指标并计算状态
4. HostsPage (前端)
   ↓ 渲染状态徽章和指标
5. 用户界面
```

## 使用方法

### 查看主机状态
1. 访问主机管理页面
2. 查看"状态"列的徽章
3. 将鼠标悬停在状态徽章上查看详细信息

### 状态徽章说明
- **✓ 正常** - 所有指标正常
- **⚠ 异常** - 部分指标接近阈值
- **✗ 严重** - 部分指标超过阈值
- **○ 离线** - 连接失败或无数据

### 指标颜色说明
- **黑色** - 正常范围 (< 80%)
- **黄色** - 警告范围 (80-89%)
- **红色** - 危险范围 (>= 90%)

## 阈值配置

当前阈值硬编码在代码中，可根据需要调整：

### 警告阈值 (Warning)
- CPU: 80%
- 内存: 80%
- 磁盘: 80%

### 严重阈值 (Error)
- CPU: 90%
- 内存: 90%
- 磁盘: 90%

### 离线阈值 (Offline)
- 数据新鲜度: 5分钟

## 测试验证

### 测试场景

1. **正常状态**
   - 所有指标 < 80%
   - 应显示绿色"正常"徽章

2. **警告状态**
   - CPU 85%, 内存 75%, 磁盘 70%
   - 应显示黄色"异常"徽章
   - 悬停显示"CPU使用率较高 (85.0%)"

3. **严重状态**
   - CPU 95%, 内存 92%, 磁盘 88%
   - 应显示红色"严重"徽章
   - 悬停显示"CPU使用率过高 (95.0%)；内存使用率过高 (92.0%)"

4. **离线状态**
   - 超过5分钟未收到数据
   - 应显示灰色"离线"徽章
   - 悬停显示"连接失败（超过5分钟未收到数据）"

5. **无数据状态**
   - 新添加的主机，尚未收集数据
   - 应显示灰色"离线"徽章
   - 悬停显示"暂无监控数据"
