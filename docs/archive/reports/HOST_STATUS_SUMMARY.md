# 主机状态显示功能实现完成

## 功能概述

为主机管理列表添加了实时状态显示功能，可以直观地查看每台主机的运行状态和资源使用情况。

## 实现内容

### ✅ 状态类型

1. **✓ 正常** (绿色)
   - 所有指标 < 80%
   - 最近5分钟内有数据更新

2. **⚠ 异常** (黄色)
   - CPU/内存/磁盘任一指标 >= 80% 且 < 90%
   - 悬停显示具体异常项

3. **✗ 严重** (红色)
   - CPU/内存/磁盘任一指标 >= 90%
   - 悬停显示具体严重问题

4. **○ 离线** (灰色)
   - 超过5分钟未收到数据
   - 或从未收到过数据

### ✅ 显示内容

**表格列**：
- 名称
- 主机 IP
- 端口
- 用户名
- **状态**（新增，带悬停提示）
- **CPU**（新增，带颜色标识）
- **内存**（新增，带颜色标识）
- **磁盘**（新增，带颜色标识）
- 操作

**指标颜色**：
- 正常 (< 80%): 默认颜色
- 警告 (80-89%): 黄色
- 危险 (>= 90%): 红色

## 状态判断逻辑

```python
# 1. 检查数据新鲜度
if metric_age > 300:  # 5分钟
    status = "offline"

# 2. 检查指标阈值
elif any(metric >= 90):
    status = "error"  # 严重

elif any(metric >= 80):
    status = "warning"  # 异常

else:
    status = "normal"  # 正常
```

## 示例场景

### 场景1：正常主机
- CPU: 45%, 内存: 60%, 磁盘: 55%
- 显示：✓ 正常（绿色）
- 提示：运行正常

### 场景2：CPU 高负载
- CPU: 85%, 内存: 60%, 磁盘: 55%
- 显示：⚠ 异常（黄色）
- 提示：CPU使用率较高 (85.0%)

### 场景3：多项指标异常
- CPU: 95%, 内存: 92%, 磁盘: 88%
- 显示：✗ 严重（红色）
- 提示：CPU使用率过高 (95.0%)；内存使用率过高 (92.0%)

### 场景4：连接失败
- 超过5分钟未收到数据
- 显示：○ 离线（灰色）
- 提示：连接失败（超过5分钟未收到数据）
