# 资源大盘重新设计规格文档

**日期**：2026-03-22
**状态**：已确认，待实现

---

## 概述

对 DbGuard 资源大盘（Dashboard）进行全面视觉重设计，目标：
- 深色科技感，深蓝专业风 + 玻璃拟态效果
- 一眼看出全部主机和数据库的整体运行状态
- 异常资源自动置顶并高亮显示

---

## 视觉风格

| 属性 | 值 |
|------|----|
| 背景 | `#0d1117` → `#0a1628` 渐变，极淡网格纹理 |
| 主色调 | 蓝白系，`#58a6ff`（信息蓝）、`#3fb950`（健康绿）、`#f85149`（异常红）、`#d29922`（警告橙） |
| 卡片背景 | `rgba(255,255,255,0.04)` + `backdrop-filter: blur(12px)` |
| 卡片边框 | `1px solid rgba(255,255,255,0.08)` |
| 字体 | 系统字体，数字/指标使用等宽字体（`font-variant-numeric: tabular-nums`） |

## 组件设计

### 顶部三栏概览

三栏等宽排列，背景为玻璃拟态卡片，高度约 200px，用 `1px solid rgba(255,255,255,0.06)` 分隔。

**左栏：主机健康区**
- 数据来源：`API.getHosts()` 返回数组，每项含 `status` 字段，值为 `normal`/`warning`/`error`/`offline`/`unknown`
- 在线判定：`status` 为 `normal` 或 `warning` 算在线；`offline` 或 `error` 算异常；`unknown` 算未知，归入在线统计但用灰色显示
- SVG 环形图（Donut Chart），绿弧=在线（含 unknown），红弧=离线/错误
- 中心显示存活率（32px 等宽字体，如 `92%`）
- 下方最多列出 3 个异常主机名（`status` 为 `offline` 或 `error`）+ 红色状态点
- 无异常时显示 `全部在线` 绿色文字

**中栏：活跃告警列表**
- 数据来源：`API.getAlerts({ status: 'active', limit: 5 })`，响应结构为 `{ alerts: [...], total: N }`，取 `response.alerts`
- 告警字段：`severity`（`critical`/`high`/`medium`/`low`）、`title`（标题）、`created_at`（时间）
- 严重度颜色：`critical`/`high` → 红色 `#f85149`；`medium` → 橙色 `#d29922`；`low` → 蓝色 `#58a6ff`
- 每条格式：`[色块] title  时间相对值（如 5m前）`
- 最多显示 5 条
- 无告警时：`✓ 系统运行正常`（绿色）
- 标题：`⚡ 活跃告警` + 数量气泡（`response.total`）

**右栏：数据库健康区**
- 数据来源：`API.getDatasourceHealth(connId)` 逐个获取，健康判定：`status === 'healthy'` 为健康，`warning` 为警告，其余为异常
- SVG 环形图显示健康率（healthy 数 / 总数）
- 下方各 DB 类型分布（色块 + 数量）：
  ```
  MySQL      ████  8    PostgreSQL  ██  3
  Redis      █     2    其他        █   1
  ```

### 资源卡片

卡片宽度：自适应网格（最小 280px），4 列为主。

**正常卡片：**
```
┌─────────────────────────────────────────┐
│  🟢  mysql-prod-01           [MySQL]   │
│  192.168.1.10:3306 / prod_db            │
│  ─────────────────────────────────────  │
│  连接数     CPU       QPS      健康      │
│  142        23.4%     1,204    ● 健康   │
│  ████░░░    ██░░░░░                     │
└─────────────────────────────────────────┘
```
- 边框：`1px solid rgba(255,255,255,0.08)`
- 背景：`rgba(255,255,255,0.04)` + blur
- Hover：上浮 2px + 边框亮度增加
- 点击跳转到监控页

**异常卡片（置顶区域）：**
```
┌─────────────────────────────────────────┐  ← 红色外发光
│  🔴  mysql-prod-02           [MySQL]   │
│  192.168.1.11:3306 / prod_db            │
│  ─────────────────────────────────────  │
│  连接数     CPU       QPS      健康      │
│  498        94.7%     --       ✗ 异常   │
│  ████████   ████████                    │  ← 进度条红色
└─────────────────────────────────────────┘
```
- critical/error：边框 `1px solid rgba(248, 81, 73, 0.6)`，发光 `box-shadow: 0 0 16px rgba(248, 81, 73, 0.25)`
- warning：橙色边框 `rgba(210, 153, 34, 0.6)`，橙色发光

**异常置顶区标题：**
- `⚠ 需要关注 (N)` 红色文字
- 仅在存在 critical 或 warning 资源时显示该区块
- 过滤器也会影响异常置顶区（保持一致）

---

### 数字动效

- 页面加载时数字从 0 计数到实际值（JS `requestAnimationFrame` 实现，持续 600ms）
- CPU / 连接数进度条同步宽度动画（CSS `transition: width 600ms ease`）
- 顶部刷新状态：`● 实时  最后更新 14:32:05`，绿色点每 2s CSS `opacity` 闪烁

---

### 状态处理

| 状态 | 处理方式 |
|------|----------|
| 加载中 | 保留现有 `loading-overlay` + spinner，三栏概览用骨架占位（灰色矩形） |
| 空状态（无数据源） | 保留现有空状态引导页 |
| 过滤后无结果 | 正常区显示 `没有符合条件的资源`，异常区隐藏 |
| API 出错 | 顶部三栏对应区显示 `--`，卡片指标显示 `--` |

---

## 过滤器行为

- 现有过滤器（健康状态、DB类型、主机、搜索）保留在 Header 区域，样式跟随深色主题更新
- 过滤器同时作用于**异常置顶区**和**正常资源区**（两者保持一致）
- 顶部三栏概览始终统计**全量数据**，不受过滤器影响

## 响应式布局

| 断点 | 布局变化 |
|------|----------|
| ≥1200px | 三栏等宽，资源卡片 4 列 |
| 768px–1200px | 三栏保持，资源卡片 2–3 列 |
| <768px | 三栏垂直堆叠，资源卡片 1 列，过滤器换行 |

## 数据来源（与现有代码对接）

| 数据 | API 方法 | 说明 |
|------|----------|------|
| 数据源列表 | `API.getDatasources()` | 返回数组 |
| 主机列表 | `API.getHosts()` | 每项含 `status` 字段 |
| 数据库健康状态 | `API.getDatasourceHealth(connId)` | 逐个获取，`status` 字段 |
| 最新指标 | `API.getLatestMetric(connId)` | 含 CPU、连接数、QPS |
| 活跃告警 | `API.getAlerts({ status: 'active' })` | 响应为 `{ alerts: [...], total: N }` |

刷新频率保持 15s 不变。
