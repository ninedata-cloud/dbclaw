# 数据源临时静默功能 - 快速使用指南

## 功能说明

当数据库需要临时关闭或维护时，可以设置静默期，在此期间系统将：
- 停止指标采集
- 停止告警通知
- 停止巡检触发

静默期结束后自动恢复监控。

## 使用步骤

### 1. 设置静默

1. 进入"数据源管理"页面
2. 找到需要静默的数据源
3. 点击"临时静默"按钮（静音铃铛图标）
4. 在弹出对话框中：
   - 选择静默时长（1-72小时）
   - 填写静默原因（可选，建议填写）
5. 点击"确认静默"

### 2. 查看静默状态

在数据源列表的"状态"列可以看到：
- **监控中**: 绿色徽章 + 铃铛图标
- **静默中**: 橙色徽章 + 静音铃铛图标 + 剩余时长

### 3. 取消静默

1. 找到已静默的数据源
2. 点击"取消静默"按钮（橙色铃铛图标）
3. 确认后立即恢复监控

### 4. 自动恢复

静默期结束后，系统会自动：
- 清除静默配置
- 恢复指标采集
- 恢复告警通知

## 常见场景

### 场景1: 数据库维护（4小时）

```
操作: 设置静默 → 选择4小时 → 填写"数据库维护"
结果: 4小时内不采集、不告警，4小时后自动恢复
```

### 场景2: 系统升级（提前结束）

```
操作: 设置静默24小时 → 升级完成后手动取消静默
结果: 升级期间不告警，完成后立即恢复监控
```

### 场景3: 临时关闭数据库

```
操作: 关闭前设置静默 → 避免连接失败告警
结果: 不会收到大量告警通知
```

## 注意事项

1. **静默时长**: 最长72小时
2. **静默原因**: 建议填写，便于后续审计
3. **手动取消**: 可随时提前取消静默
4. **自动恢复**: 静默期结束后自动恢复，无需手动操作

## API 接口

如需通过API操作，可使用以下接口：

```bash
# 设置静默
POST /api/datasources/{id}/silence
{
  "hours": 2,
  "reason": "数据库维护"
}

# 取消静默
DELETE /api/datasources/{id}/silence

# 查询静默状态
GET /api/datasources/{id}/silence
```

## 验证方法

### 验证不采集

1. 设置静默
2. 等待60秒（采集周期）
3. 查看监控页面，确认无新数据

### 验证不告警

1. 设置静默
2. 触发告警条件（如关闭数据库）
3. 确认不收到告警通知

### 验证自动恢复

1. 设置短时静默（1小时）
2. 等待静默期结束
3. 确认自动恢复监控和告警
