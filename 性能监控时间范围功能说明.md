# 性能监控时间范围功能说明

## 功能概述

为性能监控页面添加了时间范围选择功能，支持查看不同时间段的历史监控数据。

## 功能特性

### 1. 预设时间范围

在页面顶部提供时间范围下拉选择器，包含以下预设选项：

- **最近 1 分钟**
- **最近 10 分钟**
- **最近 1 小时**（默认）
- **最近 6 小时**
- **最近 1 天**
- **最近 7 天**
- **最近 1 个月**
- **自定义时间**

### 2. 自定义时间范围

选择"自定义时间"后，会弹出对话框，允许用户：

- 选择开始时间（datetime-local 输入框）
- 选择结束时间（datetime-local 输入框）
- 验证时间范围的有效性（开始时间必须早于结束时间）

### 3. 实时监控控制

新增"实时监控"按钮，用于控制实时数据更新：

- **实时监控**（蓝色按钮）：启用 WebSocket 实时推送
- **暂停实时**（灰色按钮）：停止 WebSocket 连接，仅显示历史数据

### 4. 数据加载优化

- 切换时间范围时自动清空图表并重新加载数据
- 支持最多 10,000 条历史记录查询
- 批量更新图表以提升性能

## 后端 API 改进

### `/api/metrics/{conn_id}` 端点增强

新增查询参数：

```python
@router.get("/{conn_id}", response_model=List[MetricResponse])
async def get_metrics(
    conn_id: int,
    metric_type: Optional[str] = None,
    start_time: Optional[datetime] = None,  # 新增：开始时间
    end_time: Optional[datetime] = None,    # 新增：结束时间
    minutes: Optional[int] = None,          # 新增：最近N分钟
    limit: int = Query(1000, le=10000),     # 提升：最大10000条
    db: AsyncSession = Depends(get_db),
)
```

**参数说明**：

- `minutes`: 查询最近 N 分钟的数据（优先级最高）
- `start_time`: 自定义开始时间（ISO 格式）
- `end_time`: 自定义结束时间（ISO 格式）
- `limit`: 最大返回记录数（默认 1000，最大 10000）

**使用示例**：

```bash
# 查询最近 60 分钟
GET /api/metrics/1?metric_type=db_status&minutes=60

# 查询自定义时间范围
GET /api/metrics/1?metric_type=db_status&start_time=2026-03-15T10:00:00&end_time=2026-03-15T12:00:00

# 查询最近 1 天（最多 10000 条）
GET /api/metrics/1?metric_type=db_status&minutes=1440&limit=10000
```

## 前端实现细节

### 状态管理

```javascript
const MonitorPage = {
    currentTimeRange: 60,      // 当前时间范围（分钟）
    isRealtime: true,          // 是否启用实时监控
    // ...
}
```

### 核心方法

1. **`_reloadData(connId)`**: 重新加载数据
   - 停止 WebSocket
   - 清空所有图表
   - 重置网络追踪状态
   - 加载新的历史数据
   - 根据 `isRealtime` 决定是否启动 WebSocket

2. **`_showCustomTimeDialog()`**: 显示自定义时间对话框
   - 创建模态对话框
   - 提供开始/结束时间选择器
   - 验证时间范围有效性
   - 调用 `_loadCustomRange()` 加载数据

3. **`_loadCustomRange(connId, startTime, endTime)`**: 加载自定义时间范围数据
   - 转换为 ISO 格式
   - 调用 API 获取数据
   - 清空图表并重新渲染

4. **`_stopMonitoring()`**: 停止 WebSocket 监控
   - 断开 WebSocket 连接
   - 清理资源

## 使用流程

1. **查看实时数据**（默认）
   - 页面加载后自动显示最近 1 小时数据
   - WebSocket 实时推送新数据

2. **切换时间范围**
   - 从下拉菜单选择预设时间范围
   - 自动重新加载对应时间段的数据
   - 保持实时监控状态

3. **查看历史数据**
   - 选择"自定义时间"
   - 在对话框中选择开始和结束时间
   - 点击"确定"加载数据
   - 自动暂停实时监控

4. **恢复实时监控**
   - 点击"暂停实时"按钮切换为"实时监控"
   - 重新启动 WebSocket 连接

## 注意事项

1. **数据量限制**：单次查询最多返回 10,000 条记录
2. **时间验证**：自定义时间范围时，开始时间必须早于结束时间
3. **实时状态**：选择自定义时间后会自动暂停实时监控
4. **性能优化**：使用批量更新减少图表重绘次数
